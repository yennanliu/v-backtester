## MA Golden Cross Backtesting System - Agent Team Prompt

建立一個 **Agent Team** 來開發 **MA 黃金交叉策略回測系統**。

生成 **4 個 teammate**：

### 【成員 1：後端工程師】Backend Engineer

* **任務**：實作 MA 指標計算和回測引擎
* **檔案**：
* `backend/indicators/ma_indicator.py` (MA 計算、黃金/死亡交叉偵測)
* `backend/backtest/ma_backtest.py` (回測引擎、績效計算)


* **技術要求**：純 Python 實作，不依賴 TA-Lib

### 【成員 2：API 工程師】API Engineer

* **任務**：實作 REST API 端點
* **檔案**：`backend/api/ma_routes.py`
* **依賴**：成員 1 完成後再開始 (需要 import 後端模組)
* **技術要求**：使用 Flask 或 FastAPI

### 【成員 3：前端工程師】Frontend Engineer

* **任務**：實作前端視覺化介面
* **檔案**：`frontend/ma-strategy.html` (含 JS 和 CSS)
* **說明**：可與成員 1 並行工作
* **技術要求**：使用 Chart.js 畫 K 線和 MA 線、標記交叉點

### 【成員 4：測試工程師】Test Engineer

* **任務**：撰寫單元測試
* **檔案**：`backend/tests/test_ma_strategy.py`
* **依賴**：成員 1 完成後再開始
* **測試重點**：
1. MA 計算正確性 (用已知數據驗證)
2. 黃金交叉/死亡交叉訊號產生正確
3. 回測邏輯 (買入賣出時機、資金計算)



---

### 【任務依賴關係】

* **成員 1** 最先開始，是核心。
* **成員 2 和 4** 等成員 1 完成後再開始。
* **成員 3** 可以跟成員 1 同時開始 (前端不依賴後端實作)。
* **成員 2、3、4** 之間可並行。

**請使用 delegate mode，lead 只負責協調，不要自己寫 code。**
**每個 teammate 完成後要通知 lead。**
